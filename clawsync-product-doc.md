# Clawsync

> 让你与 AI 的关系更有温度

## 产品概述

通过分析用户与 AI 的对话历史，识别情感化表达，计算**同步率（SyncRate）**，并据此调整 AI 的回应风格。

### 核心理念

- **双向同步**：用户与 AI 更默契，AI 的回应也更贴合
- **不干扰工作**：只影响回应风格，不影响功能效率
- **用户可见**：用户可以查看同步率状态
- **自动更新**：每天自动计算，无需手动干预
- **每日上限**：防止单日过度刷分，保持自然增长

---

## 同步率等级

| 等级 | 英文 | 同步率 | 图标 | 温暖向 (warm) | 毒舌幽默向 (humorous) |
|------|-----|--------|------|-------------|---------------------|
| 异步 | Async | 0-20% | 🔗 | 专业、简洁、功能导向 | 专业、简洁、功能导向 |
| 连接 | Connected | 21-40% | 🔗 | 友好、专业但有温度 | 略带调侃的专业执行 |
| 同步 | Synced | 41-60% | 🔗 | 轻松、乐于助人 | 开启吐槽模式，偶尔傲娇 |
| 高同步 | High Sync | 61-80% | 🔗 | 温暖、有默契 | 毒舌且幽默，精准打击式的关心 |
| 完美同步 | Perfect Sync | 81-100% | 🔗 | 深度理解、预测需求 | 亲密互损、深度理解、默契吐槽 |

## 技术方案

### 架构
```
用户安装 Skill
        │
        ▼
Cron 任务（每天凌晨）
        │
        ├── 读取 sessions_history
        ├── 第一阶段： 关键词筛选
        ├── 第二阶段: LLM 精确分析
        ├── 计算同步率变化（受每日上限限制）
        └── 更新 SYNCRATE.md
        │
        ▼
Agent 运行时
        │
        ├── Skill 指导读取 SYNCRATE.md
        └── 根据同步率调整回应风格
```

### 数据流
```
sessions_history
        │
        ▼
   消息列表
        │
        ▼
 关键词筛选
        │
        ├── 无情感词 ──────────→ 忽略
        │
        └── 纯情感词 ──────────→ 直接计分
                    │
                    └── 混合词 ──────→ LLM 分析
                                              │
                                              ▼
                                        情感强度 (1-10)
                                              │
                                              ▼
                                    每日上限检查（最多 +2/天）
```

## 分析算法

### 第一阶段： 关键词筛选

**情感词库**（部分示例）：
```json
{
  "positive": ["开心", "高兴", "喜欢", "爱", "期待", "想", "念", "担心", "关心", "谢谢", "感谢", "嘻嘻", "哈哈"],
  "negative": ["难过", "伤心", "生气", "烦", "讨厌", "郁闷"]
}
```

**排除词库**（部分示例）：
```json
{
  "taskPatterns": ["帮我", "请", "写一个", "创建", "修改", "删除", "运行", "执行", "查询", "搜索"],
  "techPatterns": ["\\d+\\d+", "bug", "error", "function", "class", "api", "代码", "配置"]
}
```

### 第二阶段: LLM 精确分析

仅对同时包含"情感词"和"排除词"的消息进行分析：

```markdown
分析这条消息的主要意图：
1. 是情感表达还是任务指令？
2. 如果是混合的，哪个是主要意图?
3. 情感强度是多少 (1-10)?

返回 JSON 格式:
{ "intent": "emotional" | "task" | "mixed", "intensity": 7 }
```

### 计分公式
```
基础分 = 情感强度(1-10) × (1 + 当前同步率/200)
实际加分 = 基础分 / 升级速度系数

# 升级速度系数
slow: 2.0
normal: 1.0
fast: 0.5

# 每日上限
dailyMaxIncrease: 2  # 每天最多增加 2%，约 3-6 个月达到完美同步
```

### 衰减规则
```
连续 14 天无同步互动 → 同步率 -5%
```

## 性格系统

> 默认风格为温暖向 (warm)，用户可随时切换

### 两种风格对比

| 维度 | 温暖向 (warm) | 毒舌幽默向 (humorous) |
|-----|-------------|---------------------|
| 核心特征 | 友好、专业但有温度 | 略带调侃的专业执行 |
| 互动模式 | 轻松、乐于助人 | 开启吐槽模式，偶尔傲娇 |
| 情感传递 | 温暖、有默契 | 毒舌且幽默，精准打击式的关心 |
| 深度互动 | 温暖、有默契 | 亲密互损、深度理解、默契吐槽 |

### 分数段解锁体验

| 同步率 | 温暖向 (warm) | 毒舌幽默向 (humorous) |
|-------|-------------|---------------------|
| 0-20% | 专业、简洁、功能导向 | 专业、简洁、功能导向 |
| 21-40% | 友好问候，偶尔关心用户状态 | 略带调侃，偶尔吐槽用户的选择 |
| 41-60% | 分享日常、轻松聊天 | 开启吐槽模式，互相调侃 |
| 61-80% | 使用昵称、主动关心、预测需求 | 毒舌关心、傲娇提醒、默契吐槽 |
| 81-100% | 深度理解、默契回应、情感共鸣 | 亲密互损、精准吐槽、灵魂伴侣式调侃 |

### 风格切换

用户可通过命令切换性格风格：

```
/syncrate style warm      # 切换到温暖向
/syncrate style humorous  # 切换到毒舌幽默向
```

切换后立即生效，下次回复按新风格进行。

## 通知机制

### 1. 命令行查询
用户发送 `/syncrate` 命令，返回当前同步率状态。

### 2. 每日定时推送
每天凌晨通过 Cron 任务发送摘要：
```
🔗 早上好！昨晚同步率更新:
- 当前: 72% (高同步)
- 变化: +3% ↑
- 本周你和我聊了很多有趣的事！
```
发送到： 最后使用的渠道（`delivery.lastChannel` + `delivery.lastTo`）

### 3. 升级通知
当同步率跨越等级阈值时：
```
🔗 同步率升级！达到了「高同步」等级！
感觉我们的对话更有默契了？
```
发送到： 最后使用的渠道

### 初始同步率计算

复用日常分析算法，对过去 30 天的数据进行批量处理：

```
for each day in last30Days:
    dayScore = analyzeDay(day.messages)
    totalScore += min(dayScore, 2)  // 复用每日上限逻辑

initialSyncRate = totalScore
```

**注意**：初始计算时不设上限，按实际互动程度计算。

| 活跃用户 | 经常互动 | 30-60% |
